# Historical Reports Archive

This directory contains historical documentation that represents snapshots from specific points in time during the development of paku-iot.

## Purpose

These documents are preserved for:
- Historical reference and context
- Understanding implementation decisions
- Tracking project evolution
- Debugging and troubleshooting past issues

## Contents

- **Status Reports**: Point-in-time system status snapshots (e.g., `SYSTEM_STATUS.md`, `ECOFLOW_STATUS.md`)
- **Fix Summaries**: Records of specific bug fixes and issues resolved
- **Completion Reports**: Documentation of completed features and milestones
- **Analysis Documents**: Technical analysis and decision-making documentation

## Current Documentation

For up-to-date documentation, please refer to:
- [Main README](../../README.md) - Project overview and quick start
- [docs/](../../docs/) - Current technical documentation
- Component READMEs in [stack/](../../stack/) - Service-specific documentation

## Note

All documents in this archive include a header clearly marking them as historical. They may contain outdated information and should not be used as the primary reference for current system behavior.
